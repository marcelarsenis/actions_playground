# DBT Failure Webhook Test Case - Issue #14

## Overview

This document describes **Issue #14**, which serves as a test case for the dbt failure agent webhook system.

## Issue Details

- **Issue Number**: #14
- **Title**: [Auto] Production Daily Build failed: base_gong__gong_usage [000904]
- **Source**: Auto-generated by `dbt-failure-agent` from dbt Cloud webhook event `job.run.errored`
- **Status**: Auto-resolved (next run succeeded)

## What This Test Demonstrates

This issue demonstrates the complete dbt failure webhook pipeline:

1. **Webhook Trigger**: When a dbt Cloud job fails, it sends a webhook to the Azure Function
2. **Issue Creation**: The Azure Function creates a GitHub Issue with structured context:
   - Job details (id, name, run id, etc.)
   - Failed models and their error messages
   - Source table schema information
   - Recommended validation steps
3. **Agent Assignment**: The issue can be assigned to GitHub Copilot Coding Agent for automatic fixes
4. **Auto-Resolution**: When the external dbt job succeeds on retry, this indicates the issue was transient
